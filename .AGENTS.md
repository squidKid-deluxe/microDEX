# microDEX Architecture Guide

> This document is for AI agents navigating the codebase. It describes the structure, patterns, and conventions used in this project.

## Project Overview

**microDEX** is a real-time cryptocurrency trading interface for the **BitShares** blockchain. It provides a minimalist, zero-framework UI for viewing order books, trade history, and executing trades on the BTS:HONEST.MONEY market pair.

- **Version:** 1.0.0
- **License:** WTFPL
- **Frontend only:** Vanilla HTML/CSS/JavaScript — no build tools, no backend, no Python
- **Data source:** Direct WebSocket connections to BitShares public nodes via `GrapheneRPCPool`
- **Trading pair:** Configurable via Settings UI (defaults: `BTS:HONEST.MONEY`, account `squidkid-deluxe256`)

## Architecture

```
┌─────────────────────────────────────────────────────┐
│                    Browser (microDEX)               │
│                                                     │
│  ┌───────────────┐    ┌──────────────────────────┐  │
│  │ GrapheneRPCPool ├──▶ BitShares Node 1 (wss)   │  │
│  │ (read: book,  │    ├──────────────────────────┤  │
│  │  history,     │    │ BitShares Node 2 (wss)   │  │
│  │  balances)    │    ├──────────────────────────┤  │
│  │               │    │ BitShares Node N (wss)   │  │
│  └───────┬───────┘    └──────────────────────────┘  │
│          │  failover + health tracking               │
│          ▼                                           │
│  ┌───────────────┐    ┌──────────────────────────┐  │
│  │ bitshares_js  │◀───▶ BitShares Node (wss)     │  │
│  │ (write: buy,  │    (connected via             │  │
│  │  sell, cancel)│     wss_handshake before tx)  │  │
│  └───────────────┘    └──────────────────────────┘  │
│                                                     │
│  metaNode  ←  global object updated by poll loop    │
│  cache     ←  resolved account/asset IDs/precisions │
│  pool      ←  GrapheneRPCPool instance              │
└─────────────────────────────────────────────────────┘
```

### Two WebSocket Systems

1. **`GrapheneRPCPool`** (from `graphene-rpc.js`) — used for **reading** data:
   - Order book, trade history, balances, open orders, ticker price
   - Manages a pool of 15 nodes with automatic failover, health tracking, and retry logic
   - Updated every ~2.5 seconds by the poll loop in `updater.js`

2. **`bitshares_js` library** — used for **writing** transactions:
   - Buy/sell orders, order cancellations, wallet signing
   - Connected to the pool's current node via `wss_handshake()` before each broadcast
   - Uses the user's WIF private key (client-side only, never sent anywhere)

## Key Files

### `graphene-rpc.js`
Contains two classes exposed on `window`:

**`GrapheneRPC`** — single-node WebSocket client:
- Connects to one BitShares node with timeout, reconnect, and auto-ping
- Provides methods: `rpcBook`, `rpcMarketHistory`, `rpcOpenOrders`, `rpcLast`, `rpcAccountBalances`, `rpcLookupAssetSymbols`, `rpcLookupAccounts`, `rpcBlockLatency`, `rpcPingLatency`, `getObjects`, `query`

**`GrapheneRPCPool`** — multi-node pool with failover:
- Default 15 nodes (from `public_nodes()` in the old metaNODE.py)
- Proxy-based API: call any `GrapheneRPC` method on the pool and it routes through the active node
- Automatic failover on connection errors with debounce
- In-memory health tracking (`nodeHealth` map)
- Methods: `getActiveInstance()`, `getNodeStatus()`, `switchToNextNode()`, `close()`

### `updater.js`
The main orchestrator. Responsibilities:

1. **Settings management:**
   - Glowing settings button until user saves (persists in `localStorage`)
   - Default values: `squidkid-deluxe256`, `HONEST.MONEY`, `BTS`
   - Save & Connect triggers re-bootstrap with new settings

2. **Bootstrap sequence:**
   - Create `GrapheneRPCPool`
   - Resolve asset symbols → IDs and precisions
   - Resolve account name → account ID
   - Populate the global `metaNode` object skeleton
   - Connect `bitshares_js` WebSocket via `wss_handshake()`

3. **Poll loop** (every 2.5s):
   - `rpcBlockLatency` → ping, blocktime
   - `rpcLast` → last price
   - `rpcBook` → orderbook (bidp, bidv, askp, askv)
   - `rpcMarketHistory` → recent fills (3 days)
   - `rpcOpenOrders` → user's open orders
   - `rpcAccountBalances` → asset, currency, BTS balances
   - Computes derived fields: buy/sell order totals, holdings, max, invested/divested %

4. **DOM updates:**
   - `clock()` — blocktime and latency display
   - `updateOrders()` — order book tables, open orders, fill history, balance bar
   - `updateNodeScroll()` — animated scrolling node list

### `signing.js`
Wallet and transaction operations:
- `unlock()` / `checkLoggedIn()` — WIF input, key derivation, wallet state
- `createOrder(price, amount, expiration, op)` — build and broadcast limit_order_create
- `cancelOrders(orderIds)` — build and broadcast limit_order_cancel
- `broadcastTransaction(tr, type)` — reconnects to pool's current node, signs, broadcasts
- `wss_handshake(node)` — connects bitshares_js library to a specific node

### `callbacks.js`
Thin UI wrappers:
- `buy()` — reads buy_price/buy_amt inputs, calls `createOrder`
- `sell()` — reads sell_price/sell_amt inputs, calls `createOrder`
- `cancelAll()` — maps `metaNode.orders` to IDs, calls `cancelOrders`

## Conventions & Patterns

- **No frameworks, no build tools:** React/Vue/jQuery/Bootstrap/jQuery are all absent. Vanilla DOM only.
- **No backend:** Open the HTML file in a browser. No server needed.
- **Settings in localStorage:** `microdex_settings` key stores `{ account, currency, asset }`.
- **Private keys never leave the browser:** WIF is used client-side for signing only.
- **bitshares_js reconnect before broadcast:** `broadcastTransaction` reconnects to the pool's current node to keep read/write paths aligned.
- **BitShares Object ID format:** `SPACE.TYPE.INSTANCE` — `1.2.x` = accounts, `1.3.x` = assets, `1.7.x` = limit orders.

## Important Notes for Agents

1. **No Python files remain.** The old `metaNODE.py` and `serve_metanode.py` have been deleted along with `requirements.txt` and the `pipe/` directory.
2. **`metaNode` is the global contract.** signing.js, callbacks.js, and the DOM updaters all read from this object. Keep its shape stable.
3. **Settings are user-configurable.** The defaults (squidkid-deluxe256, BTS, HONEST.MONEY) are just starting values stored in `DEFAULTS` in updater.js.
4. **The pool's node list** is in `graphene-rpc.js` `GrapheneRPCPool` constructor (15 nodes). Edit there to add/remove nodes.
5. **bitshares_js WebSocket is separate from GrapheneRPCPool.** Reads go through the pool; writes go through bitshares_js. `broadcastTransaction` bridges them by reconnecting before each broadcast.
6. **Git has no remote.** This is a local-only repository.
